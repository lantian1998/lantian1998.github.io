由于我有多种架构的设备运行 Docker（包括 x86_64 的电脑和服务器，ARM32v7 的 Tinker Board 和 ARM64v8 的树莓派 3B），我的每个 Docker 镜像都要构建多种版本。最初我给每个架构单独写一份 Dockerfile，但是很明显这样难以统一管理，在软件更新修改 Dockerfile 时经常漏改文件。之后，我用的是 Docker 的构建参数功能，即 --build-arg 参数，根据参数来调用不同的基础镜像、下载不同文件。但是这样还是有很大的局限性。首先，不同项目对不同架构的称呼是不一样的。例如，我们平时说的 x86 32 位架构，i386，就被 Go 以及众多使用 Go 的项目叫做 386。类似的，ARM32v7 也可以叫做 ARMHF，而 ARM64v8 有三种写法（ARM64v8，ARM64，AARCH64）。之前，我就不得不用 bash 脚本转换出不同的称呼组合在不同的地方使用，但是这样做就需要设置很多的变量，非常麻烦。另外，有些镜像在特定架构下需要特殊处理。例如，...
